裂缝扩展预测输入输出：

非CV

  • 一、核心输入：模型 “已知信息” 的类型与实例
    输入数据需覆盖 “裂缝现状”“载体特性”“外部作用” 三类核心信息，常见类型如下：
    • 裂缝几何与初始状态数据
      • 具体输入：裂缝初始长度、宽度、深度、尖端坐标、走向（如与岩体节理的夹角）、是否贯通
      • 数据形式：数值（如长度 5cm、宽度 0.2mm）、坐标序列（如裂缝轮廓的 (x,y,z) 点集）、二值化图像（裂缝区域标记为 1，背景为 0）
    • 岩土 / 结构力学参数数据
      • 具体输入：弹性模量、抗拉强度、泊松比、断裂韧性（KIC）、岩体节理密度 / 间距、土体含水率
      • 数据形式：数值（如弹性模量 30GPa）、归一化参数（如相对含水率 0.6）、统计分布数据（如节理间距的均值 / 方差）
    • 外部荷载与环境工况数据
      • 具体输入：施加应力（拉 / 压 / 剪应力大小与方向）、荷载加载速率、温度变化、降雨量、施工扰动强度
      • 数据形式：时序数值（如每小时应力值序列）、恒定值（如均布压力 1MPa）、分级数值（如荷载等级 1-5 级）
    • 监测时序数据（动态预测用）
      • 具体输入：历史裂缝长度 / 宽度变化序列（如过去 7 天每天的测量值）、应力监测时序、位移监测数据（如裂缝尖端位移）
      • 数据形式：时间序列数组（如 [0.1mm, 0.15mm, 0.22mm]）、多维度时序矩阵（如同时包含应力 + 位移的时序）
  • 二、核心输出：模型 “预测结果” 的类型与实例
    输出需是 “可直接应用的量化指标”，对应裂缝扩展的关键预测目标，常见类型如下：
    • 扩展轨迹预测（最核心输出之一）
      • 具体输出：裂缝未来延伸的坐标路径（如二维 (x,y) 或三维 (x,y,z) 序列）、扩展方向角（如与水平方向夹角 30°）
      • 应用场景：判断裂缝是否会贯通关键结构（如隧道衬砌、坝体防渗层）
    • 扩展速度与时序预测
      • 具体输出：单位时间裂缝长度 / 宽度增长率（如 0.3mm/h）、未来特定时间点的裂缝长度（如 72 小时后长度 8cm）、扩展阶段划分（缓慢扩展 / 快速失稳）
      • 数据形式：数值、时序数组（如未来 10 个时间步的长度预测值）、分类标签（如 “稳定”“预警”“失稳”）
    • 扩展规模与极限状态预测
      • 具体输出：裂缝最终稳定长度 / 宽度、是否会形成贯通裂缝（二分类：是 / 否）、极限荷载阈值（如应力达到 5MPa 时裂缝贯通）
      • 数据形式：数值、布尔值、阈值区间（如极限应力 4.5-5.2MPa）
    • 失稳风险与概率预测
      • 具体输出：裂缝导致岩土体 / 结构失稳的概率（如 35%）、风险等级（低 / 中 / 高）、失稳发生的时间窗口（如未来 24-48 小时）
      • 数据形式：概率值（0-1）、分类标签（3 类 / 5 类风险等级）、时间区间

CV

  • 一、CV 场景下的核心输入（均为图像相关数据）
    输入本质是 “记录裂缝当前状态 + 环境信息” 的图像或图像衍生特征，常见类型及实例：

    • 初始裂缝图像数据（静态预测核心输入）
      • 具体形式：含裂缝的原始图像（如岩体表面 RGB 图、混凝土结构灰度图、CT 扫描切片图、无人机倾斜摄影图）、裂缝分割掩码（二值图：裂缝像素 = 1，背景 = 0）
      • 实例：256×256 像素的岩石表面 RGB 图、128×128 的 CT 切片灰度图、U-Net 分割后的裂缝掩码图
    • 时序裂缝图像数据（动态预测必备输入）
      • 具体形式：同一视角下的连续图像序列（如每隔 1 小时拍摄的隧道衬砌图像、加载试验中逐帧采集的岩样图像）
      • 实例：过去 5 天每天 10:00 拍摄的边坡裂缝图像序列（共 5 张）、岩石单轴压缩试验中每秒 1 帧的图像序列（共 100 帧）
    • 多模态图像数据（提升预测精度）
      • 具体形式：融合不同类型图像（如可见光 + 热成像图、RGB + 激光雷达点云渲染图、表面图像 + 内部 CT 切片图）
      • 实例：边坡裂缝的可见光图像 + 热成像图（识别隐蔽裂缝）、混凝土坝体的 RGB 图 + 雷达扫描成像图
    • 图像提取的特征数据（轻量化输入）
      • 具体形式：从图像中预处理提取的特征向量 / 矩阵（如裂缝边缘梯度特征、纹理特征、形态学参数（长度 / 宽度 / 走向）的图像编码）
      • 实例：用 CNN 提取的 256 维裂缝图像特征向量、基于霍夫变换提取的裂缝直线特征参数

  • 二、CV 场景下的核心输出（均为图像关联结果）
    输出是 “在图像上可定位、可量化” 的裂缝扩展结果，直接对应工程需求，常见类型及实例：

    • 扩展轨迹输出（最核心）
      • 具体形式：图像坐标系下的裂缝未来延伸路径（如二维像素坐标序列、图像中的折线 / 曲线）、扩展方向的图像标注（如箭头指示）
      • 实例：(x1,y1)→(x2,y2)→…→(xn,yn) 的像素坐标序列（对应图像上的裂缝延伸路线）、在原始图像上叠加的未来 30 天裂缝轨迹标注图
    • 尺寸与时序输出（动态预测重点）
      • 具体形式：图像中裂缝的长度 / 宽度增长值（像素单位或转换后的实际单位）、时序图像中每帧的裂缝尺寸预测值、图像上的尺寸变化标注
      • 实例：未来 7 天每天的裂缝宽度预测（图像像素：5px→8px→12px，对应实际：0.5mm→0.8mm→1.2mm）、在时序图像序列上叠加的裂缝长度增长曲线
    • 状态与风险输出（工程应用核心）
      • 具体形式：图像中裂缝是否贯通的二分类结果（如贯通区域标注）、失稳风险的图像热力图（高风险区域红色标注）、扩展阶段的图像标签（如 “稳定”“预警” 区域标注）
      • 实例：混凝土坝体图像上的裂缝贯通概率热力图、边坡图像中裂缝失稳高风险区域（像素坐标范围：x50-x150,y80-y200）标注

  • 三、关键说明：

    • 输入需预处理：原始图像需先做裂缝分割（如 U-Net）、图像配准（时序图像对齐同一视角）、去噪增强，避免背景干扰预测精度；
      输出需 “图像 - 物理对齐”：预测的像素坐标 / 尺寸，需通过相机标定、比例尺换算，转换为实际工程单位（如像素→mm/m），才能直接应用；
      模型选型适配输入输出：静态轨迹预测用 CNN + 全连接层，动态时序预测用 ConvLSTM/Transformer，风险热力图预测用 U-Net 类分割模型。

  • 四、解释静态预测

    • 训练时，会给模型喂大量 “成对数据”：
      • 输入：过去的原始裂缝图像（比如 1000 张不同场景的裂缝照片）；
      • 标签：这些裂缝后续的实际扩展结果（比如 “这张图里的裂缝，3 天后向东北方向延伸了 5cm”“那张图的裂缝，最终没扩展，稳定了”）。
    • 模型在训练中，会自己总结规律：
      • 比如 “裂缝长度＜2cm、宽度＜0.1mm、尖端圆钝 + 岩体纹理均匀 → 未来会缓慢扩展，沿纹理方向走”；
      • 再比如 “裂缝长度＞5cm、尖端尖锐 + 岩体有明显节理 → 未来会快速扩展，沿着节理贯通”。
    • 静态预测的标签具体形式（和预测目标一一对应，全是可量化结果）
      • 轨迹类标签（对应 “扩展轨迹预测”）
        • 形式：图像坐标系下的裂缝扩展路径坐标序列、实际工程坐标系的轨迹点集
        • 实例：像素坐标 [(20,30), (45,32), (70,35), (95,33)]（对应图像上的延伸路线）、实际坐标 [(2.0m,3.0m), (4.5m,3.2m), …]
        • 标注方式：人工在原始图像上用标注工具（如 LabelMe）勾勒未来轨迹，自动生成坐标序列
      • 尺寸类标签（对应 “扩展规模 / 速度预测”）
        • 形式：裂缝最终长度 / 宽度数值、特定时间点的尺寸数值
        • 实例：最终实际长度 5.2cm、7 天后实际宽度 0.8mm、扩展速度 0.3mm/h
        • 标注方式：用图像标尺测量实际尺寸，或现场实测（如用裂缝宽度计）后记录数值
      • 状态类标签（对应 “失稳 / 贯通预测”）
        • 形式：二分类标签（0/1）、风险等级标签（1-5 级）、贯通状态标签（0 = 未贯通，1 = 贯通）
        • 实例：“失稳风险 = 2 级”“是否贯通 = 0”“扩展阶段 = 稳定”
        • 标注方式：根据现场监测结果（如是否发生滑坡、裂缝是否贯通）人工标注
      • 区域类标签（对应 “风险热力图 / 贯通区域预测”）
        • 形式：图像掩码（高风险区域像素 = 1，其他 = 0）、贯通区域的像素范围
        • 实例：边坡图像中高风险区域的二值掩码图、CT 切片中贯通裂缝的像素坐标范围
        • 标注方式：人工在图像上框选风险区域 / 贯通区域，生成掩码文件
